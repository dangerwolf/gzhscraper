# GZHScraper - 微信公众号高清图片自动化采集流水线

> 应某些网友的要求，对某个公众号里的图片进行爬取。
> 这是一个工业化的微信公众号历史文章与图片全量并发采集系统。基于“生产者-消费者”架构设计，内置 SQLite 状态机，支持断点续传、增量更新与严格的防风控机制。

## 🌟 核心特性

- **官方接口降维打击**：利用微信公众平台后台创作者接口，绕过移动端强校验，稳定获取全量历史文章。
- **全自动状态机 (断点续传)**：基于 SQLite 数据库 (`images_downloaded` 字段)，程序意外中断后重启可无缝衔接，不重复请求。
- **多线程高并发**：文章内的图片提取与下载采用 `ThreadPoolExecutor` 并发处理，极大缩短 I/O 阻塞时间。
- **动态防封禁**：内置基于人类行为模拟的随机休眠机制（Anti-ban sleep）与错误自适应捕获。
- **高清原图过滤**：精准剥离缩略图与格式化图标，专门抓取 `640` 尺寸的高清大图。

## 🛠️ 参数获取指南 (必须修改的配置)

为了突破微信极其严苛的反爬机制，您必须使用**公众号后台借道法**获取接口通行证。

### 1. 获取 `__biz` (目标公众号的唯一标识，又称 fakeid)
如果您不知道目标公众号的 `__biz`，请随便找一篇该公众号的文章链接。
运行工具脚本：
```bash
python get_history.py
```
*(注意：需要在代码底部 `test_url` 处替换为您手中的文章链接，运行后控制台会打印出类似 `MzI4MjkzMTcwNQ==` 的字符串，这就是 fakeid。)*

### 2. 获取 `Cookie` 和 `Token` (超级通行证)
1. 电脑浏览器访问 [微信公众平台](https://mp.weixin.qq.com/)，扫码登录您自己的微信公众号（如果没有，可免费注册一个个人订阅号）。
2. 在左侧菜单点击 **草稿箱 -> 新建图文**。
3. 在顶部工具栏找到并点击 **“超链接”** 图标。
4. 按 `F12` 打开浏览器的开发者工具，切换到 **Network (网络)** 面板。
5. 在超链接弹窗中，选择 **“选择其他公众号”**，输入任意词汇点击搜索。
6. 在 Network 面板中找到名为 `appmsg?action=list_ex...` 的请求，点击它：
   - 在 **Request Headers** 中找到 `Cookie`，完整复制。
   - 在 **Payload (或 URL 参数)** 中找到 `token`（一串纯数字），完整复制。

### 3. 配置 `wechat_mp_scraper.py`
将上述获取的参数，填入 `wechat_mp_scraper.py` 顶部的专家配置区：
```python
MP_COOKIE = "此处粘贴完整的Cookie字符串"
MP_TOKEN = "此处粘贴纯数字token"
TARGET_FAKEID = "此处粘贴目标公众号的__biz"
```

## 🚀 使用使用说明与先后次序

### 第一步：初始化与拉取文章列表 (生产者)
配置好参数后，运行文章抓取脚本。它会以模拟人类的翻页速度，将目标公众号的所有历史文章拉取并保存到本地 `images_list.db` 的 `article_list` 表中，状态标记为“未下载图片”。
```bash
python wechat_mp_scraper.py
```
*💡 提示：官方接口有严格频控。如果抓取几百篇后提示 `[频控触发]`，请停止运行，休息30-60分钟后再次运行，系统会自动跳过已入库的数据继续往下抓取。*

### 第二步：全自动并发下载图片 (消费者)
文章列表拉取完毕（或拉取了部分）后，直接无参数运行主程序。它会自动扫描数据库中 `images_downloaded = 0` 的文章，进入页面解析并并发下载所有大图，完成后将状态更新为 `1`。
```bash
python main.py
```
图片将默认保存在同级目录的 `downloaded_images/` 文件夹下。图片命名规范为：`文章标题_发布日期_序号.jpg`。

### 附加功能：单篇/手工模式
如果您不需要全量抓取，`main.py` 依然支持通过命令行参数进行单篇或多篇纯文本 URL 的手动抓取：
```bash
# 抓取单篇文章
python main.py -u "https://mp.weixin.qq.com/s/xxxxxx"

# 从包含多个URL的文本文件中批量抓取
python main.py -f "urls.txt"
```
*(手工模式下抓取的文章与图片记录，同样会被安全地合并到数据库中)*

## 🗄️ 数据库架构设计 (images_list.db)

系统会自动生成和维护以下表结构，无需手动创建：

1. **`article_list` (文章流水表)**
   - `id`: 主键
   - `title`: 文章标题
   - `article_url`: 文章永久链接 (UNIQUE 唯一索引，用于防重复抓取)
   - `publish_time`: 文章发布时间
   - `cover_url`: 封面图链接
   - `images_downloaded`: 核心状态机字段 (`0` = 仅入库尚未处理，`1` = 图片已全部下载完毕)

2. **`images_record` (图片明细表)**
   - `id`: 主键
   - `image_name`: 落地到本地的文件名
   - `image_url`: 微信图片源地址 (UNIQUE 唯一索引，用于断点续传)
   - `article_title`: 所属文章标题
   - `download_time`: 图片下载完成的时间戳
